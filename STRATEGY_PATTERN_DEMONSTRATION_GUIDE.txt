ğŸ¯ STRATEGY PATTERN IMPLEMENTATION
--------------------------------------------------------------------------------

1. PAYMENT VERIFICATION STRATEGIES
   --------------------------------
   Location: src/main/java/com/helpdesk/strategy/

   A. Manual Payment Strategy (ManualPaymentStrategy.java)
      - Requires staff approval for all payments
      - Accepts any valid amount (> Rs. 0)
      - Used when: payment.strategy.type=manual
      - Console Output: Shows "MANUAL PAYMENT STRATEGY" with detailed logs

   B. Automated Payment Strategy (AutomatedPaymentStrategy.java)
      - Auto-approves payments above Rs. 500
      - Requires manual review for payments below Rs. 500
      - Used when: payment.strategy.type=automated
      - Console Output: Shows "AUTOMATED PAYMENT STRATEGY" with threshold checks

2. CATEGORY VALIDATION STRATEGIES
   --------------------------------
   Location: src/main/java/com/helpdesk/strategy/

   A. Strict Category Strategy (StrictCategoryStrategy.java)
      - Does NOT allow duplicate category names
      - Enforces maximum length (50 characters)
      - Used when: category.strategy.type=strict
      - Console Output: Shows "STRICT CATEGORY STRATEGY" with validation results

   B. Lenient Category Strategy (LenientCategoryStrategy.java)
      - Allows duplicate category names with warnings
      - More flexible validation rules
      - Used when: category.strategy.type=lenient
      - Console Output: Shows "LENIENT CATEGORY STRATEGY" with warnings
